Items like paragraphs. ```HTML <p>Paragraph Text</p> ``` Lists ```HTML <ul> <li>List Item</li> <li>List Item</li> <li>List Item</li> </ul> ``` and images ```HTML <img src="http://www.fillmurray.com/300/300" alt="Placeholder Image" /> ```

## What Is CSS? CSS (Cascading Style Sheets) are the **style** of your document. They help control the look and feel of the document. You create **selectors** that target **HTML Elements** and then use **CSS Rules** to change the look and feel of your document. Selectors like: p { /* CSS Rules Here */ } Which targets all paragraphs, or: .rotator-slide { /* CSS Rules Here */ } Which targets *any* element with class="rotator-slide", or: #rotator { /* CSS Rules Here */ } Which targets *any* element with id="rotator".

CSS Rules go inside the {curly braces} that follow the **CSS Selector** and they look like this: 

 p {
	font-size: 2em;
	color: red;
}

Which will apply the font size of 2em and color of red to all paragraphs.

This is a paragraph with the styles above applied to it.

## What Is JS? JS (JavaScript) controls the **behavior** of your document. It allows you to do things such as animate items, hide or show items, react to user input and even process data and perform other general programming tasks. JS can look like this: var myVariable = "Hello World"; That line will create a variable called **myVariable** and assign it a value of **hello** (without the quotes). The structure and syntax of the language of JavaScript is much more complex than either HTML or CSS. This is because neither HTML nor CSS are full fledged programming languages. Neither of them have **variables**, **functions**, **objects** or **control structures** such as **if** statements or **loops**. JavaScript has *all* of these.

# Variables ## What *is* a variable? A variable is a storage location and an associated symbolic name (an identifier) which contains some known or unknown quantity or information, a value. ### In *English* professor. At its most basic we make a variable when we give a **name** to a **value**. So when we say: var myVariable = "Hello World"; We are taking the value ```hello``` and giving it the name ```myVariable```.
## Variable Types Variable *values* can be one of several different **types**. In this case, a value with text in it is called a **string** value. var myVariable = "Hello World"; Variables can also contain numbers, var meaningOfLife = 42; // Whole numbers are called 'integers' var valueOfPi = 3.14; // Numbers with decimal places are called 'floats' Variables that contain true / false values are called **booleans** var isJavaScriptCool = true; var isJavaScriptLame = false;
## Variable Types Variable can even hold lists of values (the list values themselves being **types** like strings or booleans) in what is called an **array** var myFavoriteThings = [ "Raindrops On Roses", "Whiskers On Kittens", ]; We will not go into an exhaustive list here but it is important that you understand that variables hold values and the values themselves have types.

Using a variable

The first time we create a variable we do so by using the JavaScript keyword var

var myVariable = "Hello World";

That line of code will create a new variable called myVariable and assign it the string value of Hello World

// Let's break this down.
	var myVariable = "Hello World";
/*	 |       |     |       └ The String Value "Hello World".
	 |       |     └ The 'assignment' opperator - A single equals sign.
	 |       └ The Variable Name "myVariable".
	 └ The 'var' keyword.
*/

The var keyword is only needed when you first create the variable. You do not need to use the var keyword again later when referencing or manipulating the variable.

Using a variable

After we have assigned the variable a value we can now use that variable the same way we would use the value. Let's take the following code:

var myVariable = "Hello World";
alert(myVariable);

That code would make an alert box like the one you will see if you click the link below

Click here to view the alert box

This uses the built in JavaScript function called 'alert', which creates a popup message that the user must dismiss or acknowledge before continuing any interaction with the page.

# JavaScript Functions if a variable is a **name** for a **value**, then a function is a **name** for a set of commands or instructions. For example: ```JAVASCRIPT function sayHello(){ /* There can be many commands here in between the two {curly braces} Let's make the first command an alert to the user. */ alert("Hello World"); } ``` This creates a new **function** called "sayHello" which, when invoked, will execute the **alert** function that comes built into javascipt. The **alert** function is a **Built In Function** and the **sayHello** function is a **User Defined Function**. The only difference between them is that you don't have to write a built in function, it comes for free as a part of JavaScript.
# JavaScript Functions Functions can be used to make your life a LOT easier once you get used to how to write them. A function should be used when you have something you need to do time and again in your project. ## Function Parameters Using the code from our previous slide: ```JAVASCRIPT function sayHello(){ alert("Hello World"); } ``` We notice that the ```sayHello()``` function has empty parenthesis () while the ```alert()``` function has the **string** value "Hello World" in the parenthesis (). This introduces an important concept with functions called **parameters**.
# JavaScript Functions ## Function Parameters **Parameters** are data that you pass to a function that becomes available for use IN that function as a variable. Let's say that rather than simply saying "Hello World", you might want to say something else, or many other things. Let's see what that might look like: ```JAVASCRIPT function sayAnything(message){ alert(message); } ``` Now if we wanted to use that code we could do this: ```JAVASCRIPT sayAnything("Check out my cool new boombox!"); ``` Which would cause the browser to pop up that message. [Or any other message we wanted to say](#example_2).
